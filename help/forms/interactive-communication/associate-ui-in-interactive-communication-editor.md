# 在互動式通訊編輯器中建立UI關聯

>[!NOTE]
>
> 互動式通訊功能在早期採用者程式中提供。 使用您的工作郵件地址寄送電子郵件至 `aem-forms-ea@adobe.com` 請求存取權限。

**關聯UI**&#x200B;是建置在互動式通訊(IC)編輯器之上的專門化簡化介面。 它專為面對客戶的專業人員（例如現場助理和服務代理）所設計，可在即時互動期間即時產生個人化、符合規範且精確的通訊。

![尋找IC檔案](/help/forms/interactive-communication/assets/associate-ui-preview.png)

## 關聯UI介面

關聯UI提供乾淨的雙面板工作區，讓您快速安全地產生通訊：

### 左側面板：資料輸入

- 關聯輸入或確認客戶特定資訊。
- 驗證、協助程式文字和必填欄位可指導正確輸入。

### 右側面板：即時預覽

- 顯示最終檔案的即時預覽。
- 當關聯填入資料時自動更新。

### 即時檔案產生

- 產生或下載完成的通訊。

## 使用者角色和責任

關聯UI由三個核心角色驅動，每個角色具有不同的職責：

### 1.管理員

負責系統設定、控管、後端整合和使用者存取。

| 責任 | 焦點 |
|---------------|-------|
| 系統組態 | 設定核心基礎結構、使用者群組、表單資料模型(FDM)、輸出。 |
| 治理與安全性 | 管理使用者許可權並確保系統相容性。 |
| 整合管理 | 維護後端整合和即時客戶資料連線。 |

### 2.作者

使用「關聯UI」來設計及管理互動式通訊。 ß

| 責任 | 焦點 |
|---------------|-------|
| IC製作與設計 | 建立版面、品牌和相容的檔案結構。 |
| 欄位設定 | 對應資料欄位，定義可編輯、強制和唯讀欄位。 |
| 發佈與啟用 | 發佈IC並共用連結以進行關聯存取。 |

### 3.建立關聯

使用關聯UI來協助客戶、更新資訊並產生相容的通訊。


| 責任 | 焦點 |
|---------------|-------|
| 資料確認 | 透過左側輸入面板填寫或驗證客戶資料。 |
| 預覽和驗證 | 使用即時預覽面板可確保準確性。 |
| 傳遞 | 產生PDF/電子郵件，並透過核准的管道傳送。 |

>[!NOTE]
>
> 關聯必須是&#x200B;**forms-associates**&#x200B;群組的一部分。

## 動態使用案例

關聯UI可支援即時、個人化的檔案產生功能，這對於有即時服務需求的產業而言至關重要。

| 產業 | 範例使用案例 |
|----------|-------------------|
| **金融服務** | 產生即時貸款確認函、風險設定檔摘要、帳戶建立。 |
| **保險** | 立即製作保險證明卡或索賠處理摘要。 |
| **醫療保健** | 使用計算過的Copay和排程建立患者治療計畫摘要。 |
| **公共部門** | 現場產生警方驗證報告、公民服務收據、申訴確認信和案件更新摘要。 |
| **政府** | 建立福利方案註冊的應用程式狀態摘要、服務核准信函和即時通訊。 |

## 啟用關聯UI工作流程

作者可以按照以下步驟設定和發佈互動式通訊(IC)以存取「關聯UI」：

>[!NOTE]
>
> 支援的關聯元件：日期欄位、數值欄位、文字欄位、日期時間欄位、日期欄位、核取方塊、選項按鈕、下拉式清單。

### 建立互動通訊

設計及設定互動式通訊，確保品牌、資料繫結、法規遵循規則及整合皆正確設定。

### 啟用關聯UI

從頂端動作列中，啟用「關聯UI」選項，讓IC可用於關聯驅動。

### 啟用元件中的關聯UI

### 設定可編輯欄位

在必填欄位區段中，啟用關聯可編輯的欄位。
設定驗證以確保精確且受控的資料輸入。

### 發佈互動通訊

完成所有設定後，發佈互動式通訊以取得安全存取。

### 與Associates共用已發佈的IC

提供發行的IC連結至「關聯」，讓他們能夠驗證、輸入客戶特定資訊，並產生具有有效輸入的最終通訊。

**關聯UI**&#x200B;可縮短結構化內容製作與即時客戶參與之間的差距。\
結合直覺式設計、強大的後端設定和嚴格的法規遵循控制，組織可以大規模提供&#x200B;**快速、精確和個人化的通訊**。
