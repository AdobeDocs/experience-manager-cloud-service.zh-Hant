# 表單體驗建立工具快速入門

>[!NOTE]
>
> **搶先體驗 (EA) 方案**&#x200B;提供表單體驗建立工具功能。若有興趣，請使用您的工作電子郵件地址寄送簡短電子郵件至 `aem-forms-ea@adobe.com`，請求存取該功能。

>[!IMPORTANT]
>
> **文件內容可能隨時變更**：此文件目前正在針對產品進行測試，可能會進行更新和修訂。隨著表單體驗建立工具在搶先體驗方案期間持續發展，相關的功能、命令和範例可能會有所變更。

本綜合指南能協助您開始使用對話式 AI 技術建立和管理表單。無論您是想要建立第一個表單的新手，還是希望善用複雜功能的進階使用者，都能夠找到詳細資訊和實際範例引導您了解表單體驗建立工具的功能。

## 先決條件和設定

### &#x200B;1. 請求存取權

表單體驗建立工具目前是以搶先體驗 (EA) 方案一部分的方式提供。若要參與並取得存取權限，您需要以下資訊：

**必要資訊**

- **IMS 組織 ID**：您的 Adobe 組織識別碼
- **方案 ID**：您在 Adobe Experience Cloud 內的特定方案識別碼
- **專案詳細資料**：時間軸、範圍和預期使用案例
- **官方工作電子郵件**：與您組織的 Adobe 帳戶關聯

**如何取得 IMS 組織 ID 和方案 ID**

關於尋找 IMS 組織 ID 和方案 ID 的詳細步驟，請參閱：

- [Adobe Experience Cloud 組織設定指南](/help/onboarding/cloud-manager-introduction.md)
- [方案和環境管理](/help/implementing/cloud-manager/getting-access-to-aem-in-cloud/program-types.md)

**請求存取權**

1. 使用上述指南收集您的 IMS 組織 ID 和方案 ID
2. 寄送電子郵件至 [aem-forms-ea@adobe.com](mailto:aem-forms-ea@adobe.com) 請求存取權限
3. 請求中需包含以下資訊：
   - 組織名稱和 IMS 組織 ID
   - 方案 ID
   - 專案時間軸和範圍
   - 預期使用案例和業務目標

>[!IMPORTANT]
>
> **限量開放方案**：表單體驗建立工具的存取權限需經內部利害關係人核准。Adobe 將根據方案產能以及與搶先體驗條件的相符程度審閱您的請求。不保證一定能獲得核准，且會取決於目前方案的可用性。

### &#x200B;2. 驗證表單是否已啟用

使用表單體驗建立工具之前，請確保[您的環境已啟用 AEM Forms](/help/forms/setup-forms-cloud-service.md)。


### &#x200B;3. 設定您的環境


- **若為 Edge Delivery Services (EDS)：**

   - [設定適用於 Edge Delivery Services 表單的環境](/help/edge/docs/forms/universal-editor/getting-started-universal-editor.md)
   - [使用 Edge Delivery Forms 範本建立新表單](/help/edge/docs/forms/universal-editor/create-forms.md)

- **若為核心元件型表單：**

   - 在您的 Adobe Experience Manager 執行個體上，前往「表單 > 表單與文件」
   - [使用核心元件範本建立新頁面](/help/forms/creating-adaptive-form-core-components.md)


## 快速開始

### 存取表單體驗建立工具

表單體驗建立工具可在表單管理使用者介面、通用編輯器和自適應表單編輯器中使用。您可以使用以下任何一種方法存取表單：

**表單管理使用者介面 (用於核心元件)**

1. **導覽至表單**：前往「AEM > 表單 > 表單與文件」
1. 按一下工具列中的「表單體驗建立工具」圖示。該圖示位於使用者介面的左上角一帶。
   ![AI 助理圖示*](/help/edge/docs/forms/assets/forms-manager.gif){width="50%"}
1. 開始您的對話式表單建立過程


**自適應表單編輯器 (用於核心元件)**

1. 前往「AEM > 表單 > 表單與文件」
2. [使用核心元件範本建立新表單](/help/forms/creating-adaptive-form-core-components.md)
3. 開啟您的表單進行編輯
4. 按一下編輯器工具列中的「表單體驗建立工具」圖示
   ![AI 助理圖示*](/help/edge/docs/forms/assets/adaptive-forms-editor.gif){width="75%"}

5. 開始您的對話式表單建立過程


**通用編輯器 (用於 Edge Deilvery Services Forms)**

1. 依照 [Edge Delivery Services 設定指南](/help/edge/docs/forms/universal-editor/getting-started-universal-editor.md)建立您的 EDS 頁面
1. 在通用編輯器中導覽至您的 EDS 頁面
1. 在右側面板中尋找表單體驗建立工具圖示
1. 按一下開啟對話式介面

### 基本命令

| 代碼 | 用途 | 使用範例 |
|--------|---------|---------------|
| `/` | 快速動作和快速鍵 | `/create-form contact form`、`/help validation rules`、`/update-layout wizard` |
| `@` | 參照現有的表單欄位 | `@email`、`@firstName`、`Make @phoneNumber required` |
| 純文字 | 自然對話 | 「新增必填的電話號碼欄位」、「建立電子郵件驗證」 |

**特定命令範例：**

- `/create-form customer survey` - 建立新的客戶調查表單
- `/add-field @email validation` - 將驗證新增至現有的電子郵件欄位中
- `/create-rule show @spouse if @maritalStatus equals married` - 建立條件式邏輯
- `/configure-submit to email support@company.com` - 設定電子郵件提交
- `/help multi-step forms` - 取得多步驟表單建立的協助

### 成功秘訣

- **具體表達**：「新增包含驗證的必填電子郵件欄位」效果會比「新增電子郵件」更好
- **參照現有欄位**：修改表單時使用 `@fieldName`
- **尋求協助**：輸入 `/help` 然後提出您的問題
- **迭代**：為獲得最佳效果，每次只進行一項變更


## 開始建立表單的方法

### &#x200B;1. 從自然語言提示開始

以自然語言描述您的表單需求，表單體驗建立工具會產生完整的表單結構：

**範例：**

- 「建立包含個人資訊、財務詳細資料和文件上傳的貸款申請表」
- 「建置包含評分、評論和產品類別的客戶回饋表」
- 「我需要一份會議用的多步驟註冊表單，並包含支付處理」

### &#x200B;2. 匯入和轉換

將現有的表單和文件轉變為現代化的互動式體驗：

**支援的來源：**

- **PDF 表單**：上傳靜態 PDF，將其轉換為具有驗證功能的互動式數位表單。
- **螢幕擷圖或影像**：上傳紙本表單的照片，產生有效的數位版本
- **XFA 表單**：將舊版 XFA 型表單轉換為現代回應式表單

**如何匯入：**

1. 按一下表單體驗建立工具介面中的「附件」圖示
2. 上傳您的檔案 (PDF、影像、Figma 設計等)
3. 描述您的需求：
   - 「將此 PDF 表單轉換為數位版本」
   - 「建立符合此螢幕擷圖版面的表單」
   - 「根據我的 Figma 設計建置此表單」

**支援的檔案類型：**

- **影像** (PNG、JPG、GIF)：表單版面、使用者介面模型、掃描的表單、手繪草圖
- **PDF 檔案**：現有表單、規格、文件、Acroforms、XFA 表單
- **螢幕擷圖**：桌面/行動應用程式螢幕擷圖、紙本表單照片、白板草圖
- **手繪草圖**：餐巾紙草圖、透視效果、概念圖 (翻拍)

## 最佳做法

### 表單設計

- **保持簡單**：從基本欄位開始，僅在必要時增加複雜性，避免讓使用者感到不知所措
- **使用清晰的標籤**：透過描述性標籤明確呈現欄位用途，引導使用者完成表單
- **提供說明文字**：透過內容說明和範例，引導使用者完成複雜的欄位
- **徹底測試**：驗證所有使用者路徑，確保表單在所有情境下都能正常運作

### 使用者體驗

- **漸進式揭露**：根據內容顯示相關欄位，減少認知負荷並提高完成率
- **清晰導覽**：協助使用者了解其在表單中的位置以及剩餘的步驟
- **回應式設計**：確保表單適用於所有裝置和螢幕大小，實現最大程度的可存取性
- **無障礙功能**：遵循 WCAG 指引，確保身心障礙使用者也能使用表單

### 效能

- **欄位數量最佳化**：僅詢問必要資訊，減少放棄表單的情形並提高完成率
- **使用適當的驗證**：在提交前防止錯誤，提供即時回饋和指引
- **測試完成率**：透過分析和使用者回饋，監視及提高表單有效性
- **定期更新**：讓表單始終符合業務需求和使用者期望，藉以獲得最佳效能

### 品牌一致性

- **建立品牌範本**：開始建立表單之前，先準備好具有組織顏色、字體和樣式的品牌化表單範本
- **定義樣式標準**：建立一致的按鈕樣式、欄位版面和間距指引，可於提示中參照
- **使用品牌資產**：準備標誌、顏色代碼和品牌指引，以便建立表單時能輕鬆參照
- **範本資料庫**：為常見的使用案例 (聯絡、註冊、回饋) 建置品牌化表單範本集合
- **樣式提示**：納入品牌特定的說明：「按鈕使用公司藍色 (#1234AB)，字體使用公司字體 Helvetica」



## 疑難排解

| 問題 | 快速修正 |
|-------|-----------|
| **介面未載入** | 重新整理瀏覽器、檢查網路連線 |
| **命令無法運作** | 嘗試 `/help`，或使用自然語言代替 |
| **無法識別 @fieldName** | 檢查拼字、先確保欄位存在 |
| **檔案上傳失敗** | 使用 10MB 以下的 PDF/JPG/PNG |
| **表單看起來不正確** | 更具體一些：「讓其適合行動裝置使用」 |
| **提交設定失敗** | 驗證 API 認證和權限 |

**仍需要幫助嗎？**&#x200B;輸入 `/help` 然後提出您的特定問題，或聯絡系統管理員。

如需更多支援，請參閱主要的[表單體驗建立工具提示資料庫](ai-assistant-prompt-library.md)，或聯絡系統管理員尋求技術協助。
