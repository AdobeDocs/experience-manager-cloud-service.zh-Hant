# 使用通用編輯器建立回應式表單

現代的網路環境要求各種表單，在不斷擴充的裝置和熒幕大小中都能順暢運作。 從大型桌上型電腦熒幕到小巧的智慧型手機熒幕，使用者無論選擇何種裝置，都希望獲得一致、直覺式的體驗。 建立回應式表單不再是選用專案，而是提供專業、可存取、轉換最佳化數位體驗的基本需求。

Universal Editor為開發回應式表單提供了全方位的工具和方法，這些表單會聰明地適應各種熒幕維度、輸入方法和使用者內容。 本指南探討必要的技術基礎、實作策略和最佳化技術，以建立可在所有裝置上卓越執行的表單，同時維持可用性、協助工具和視覺吸引力。

建立回應式表單涉及兩個主要活動：

- **回應式測試：**&#x200B;使用裝置模擬器針對各種螢幕尺寸預覽及測試您的表單。
- **回應式設計：**&#x200B;選取並實施無縫適應不同裝置的版面配置模式。

**閱讀本指南，您將瞭解如何：**

- 在桌上型電腦、平板電腦和行動裝置上測試表單
- 為您的內容選取合適的版面配置模式
- 套用回應式設計最佳做法
- 回應式表單常見問題疑難排解
- 最佳化表單以提高在行動裝置上的效能

## 為什麼回應式表單如此重要

**使用者體驗影響：**

- 超過 60% 的使用者使用行動裝置存取表單
- 糟糕的行動體驗導致放棄填寫率增高 67%
- 回應式表單可讓完成率提升最多 25%

**商業利益：**

- 表單完成率更高
- 提升使用者滿意度
- 加強無障礙合規性
- 降低開發和維護成本

>[!TIP]
>
> **行動裝置優先方法：**&#x200B;開始時針對行動裝置設計，然後再針對大螢幕進行加強。這樣可確保核心功能在最受限的環境中也能正常運作。

## 第一部分：在不同裝置上測試表單

在不同的裝置上測試表單，可以幫助您比使用者早一步發現回應式問題並予以解決。通用編輯器提供模擬器模式來模擬各種螢幕尺寸和方向。

### 如何測試表單

**步驟 1：開啟裝置模擬器**

1. 在通用編輯器中開啟您的表單。
2. 按一下「![模擬器圖示](/help/edge/docs/forms/universal-editor/assets/emulator.png)」{height=2%,width=2%} 工具列中的&#x200B;**模擬器**&#x200B;圖示。
3. 將會出現裝置選擇器選單。

![通用編輯器回應式測試介面](/help/edge/docs/forms/universal-editor/assets/universal-editor-emulator.png)

**步驟 2：選取要測試的裝置**

- **桌上型電腦** (寬度 1200px 以上)：預設編輯視圖
- **平板電腦** (寬度 768px 至 1199px)：中等螢幕測試
- **行動裝置** (寬度 320px 至 767px)：小螢幕測試
- **自訂**：指定特定裝置的精確尺寸

**步驟 3：測試裝置方向**

使用&#x200B;**螢幕旋轉器**&#x200B;圖示來測試兩種方向：

- **縱向模式**：標準行動裝置方向
- **橫向模式**：旋轉後的平板電腦或手機視圖

### 裝置測試結果

每種裝置類型都表現出獨特的回應式行為：

| **裝置類型** | **螢幕寬度** | **檢查項目** | **常見問題** |
|-----------------|------------------|-------------------|-------------------|
| **桌上型電腦** | 1200px 以上 | 完整版面，所有功能皆可見 | 過多的空白，過寬的版面 |
| **平板電腦** | 768px 至 1199px | 元件堆疊、導覽易用性 | 尺寸不合適、觸控瞄準問題 |
| **行動裝置** | 320px 至 767px | 單欄版面、用拇指導覽 | 字體較小、按鈕間距較近 |
| **自訂** | 使用者定義 | 裝置特定要求 | 中斷點邊緣案例 |

### 測試工作流程

**對於新表單：**

1. **在桌上型電腦視圖中建置：**&#x200B;開始即提供完整功能。
2. **在平板電腦上測試：**&#x200B;檢查中等螢幕的適用情況。
3. **在行動裝置上驗證：**&#x200B;確保在小螢幕上的可用性。
4. **修復回應式問題：**&#x200B;根據需要調整版面。
5. **重新測試所有裝置：** 確認所有尺寸的裝置均已修正。

**對於現有表單：**

1. **快速檢查行動裝置的情況：**&#x200B;該表單可以在手機上使用嗎？
2. **確認發生問題的區域：**&#x200B;注意版面配置和可用性問題。
3. **系統測試：**&#x200B;全面測試每種裝置尺寸。
4. **記錄問題：**&#x200B;追蹤需要修正的問題。
5. **實施修正：**&#x200B;按部就班地解決問題。

## 第 2 部分：選取回應式版面配置模式

版面配置模式決定表單內容如何適應不同的螢幕尺寸。正確的模式可以同時改善使用者體驗和表單在行動裝置上的效能。

### 版面模式概觀

| **版面配置類型** | **最適合** | **行動裝置效能** | **複雜度** |
|---------------------|-------------------------------------------|------------------------|----------------|
| **面板版面** | 內容分類、儀表板風格的表單 | 良好 | 低 |
| **精靈版面** | 多步驟流程、複雜工作流程 | 極佳 | 中 |
| **摺疊式版面** | 常見問題集風格的內容、可選區段 | 極佳 | 低 |

### 快速決策指南

**遇到以下情況請使用面板版面：**

- 您的內容分為不同的類別
- 使用者需要同時檢視多個區段
- 內容相對簡單

**遇有以下情況請使用精靈版面：**

- 表單有多個邏輯步驟
- 您想減少認知負荷
- 行動裝置使用者是主要客群

**遇到以下情況請使用摺疊式版面：**

- 表單包含選用或次要內容
- 節約空間很重要
- 內容可以按邏輯分組

### 面板版面

「面板配置」會將相關內容整理到視覺上不同的區段中，讓使用者一次檢視多個區段。 此版面適用於擁有分類資訊的表單，這些資訊可在大型熒幕上並排顯示。

![面板版面範例](/help/edge/docs/forms/universal-editor/assets/panel-layout.png)

**回應式行為**

- **桌上型電腦（1200px以上）：**&#x200B;面板會並排顯示，或是以格線顯示，以最大化顯示效果。
- **平板電腦(768px-1199px)：**&#x200B;面板以適當的間距垂直棧疊以保持清晰度。
- **行動裝置(320px-767px)：**&#x200B;面板以單欄版面配置，區段之間有清楚的分隔，方便瀏覽。

**如何實作**

1. 將[面板元件](https://experienceleague.adobe.com/zh-hant/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/panel)新增至您的表單。
2. 將每個面板內的相關欄位分組，以維護邏輯組織。
3. 為每個面板區段指定清楚的描述性標題。
4. 確保面板之間有足夠的間距，以防止視覺混亂。

**最佳實務**

- 桌上型電腦的面板數量限製為3或4個，以免讓使用者不知所措。
- 針對每個面板使用簡潔的描述性標題，以協助使用者瞭解。
- 以邏輯方式組織面板中的欄位，將認知負載降至最低。
- 測試觸控裝置上的面板導覽，以確保所有平台上的可用性。

**常見使用案例**

- **工作應用程式：**&#x200B;個人資訊、教育、經驗及參考資料部分。
- **產品註冊：**&#x200B;面板，提供基本詳細資料、技術規格及保固資訊。
- **調查Forms：**&#x200B;人口統計群組、偏好設定、意見回饋和連絡資訊。

### 精靈版面

精靈版面配置會引導使用者進行多步驟程式，一次顯示一個區段。 此版面配置對於複雜表單特別有效，因為將流程分成多個可管理的步驟，可減少認知負載並增加完成率。

![精靈版面範例](/help/edge/docs/forms/universal-editor/assets/wizard-layout.png)

**回應式行為**

- **所有裝置：**&#x200B;維持單一步驟焦點，這是最適合行動使用者的作法。
- **步驟內容：**&#x200B;每個步驟都會回應式調整，棧疊欄位或根據熒幕大小並排排列。
- **導覽：**&#x200B;提供觸控式按鈕，具有足夠的間距，可輕鬆互動。
- **進度指示器：**&#x200B;進度列或步驟指示器會針對不同的裝置適當調整，提供完成狀態的明確回饋。

**如何實作**

1. 將[精靈元件](https://experienceleague.adobe.com/zh-hant/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/wizard)插入您的表單。
2. 將表單分成邏輯步驟（最好介於3到7個之間），讓每個步驟都重點突出，易於管理。
3. 新增進度指示器，以協助使用者瞭解其在流程中的位置。
4. 提供清晰的導覽控制項，例如「下一步」、「上一步」和「儲存」按鈕。

**行動最佳化秘訣**

- 使用大型觸控目標（高度至少為44px）作為導覽控制項，以提升協助工具。
- 確保小熒幕上的步驟指示器可見且清晰。
- 限制每個步驟的欄位數以最小化捲動並改善焦點。
- 啟用自動儲存功能，以防止使用者離開表單時遺失資料。

**最佳實務**

- 設計步驟以遵循邏輯進度，每個步驟都建立在上一個步驟上。
- 對每個步驟使用清楚的描述性標題來設定使用者期望。
- 驗證使用者在每個步驟的輸入內容，以及早擷取錯誤並減少挫敗感。
- 允許使用者向後導覽，以檢閱或編輯先前的資訊，而不會遺失資料。

**常見使用案例**

- **保險索賠：**&#x200B;事件詳細資料、證據提交、個人資訊和複查的步驟。
- **帳戶設定：**&#x200B;基本資訊、偏好設定、安全性設定和確認階段。
- **訂單處理：**&#x200B;產品選擇、送貨資訊、付款詳細資料和訂單彙總的步驟。

### 摺疊式版面

摺疊式功能表配置將內容組織成可摺疊的區段，以節省空間，最適合用於選用或次要資訊。 此版面配置對於內容可以邏輯分組且不需要一次全部顯示的表單特別有效。

![摺疊式版面範例](/help/edge/docs/forms/universal-editor/assets/accordion-layout.png)

**回應式行為**

- **行動效能：**&#x200B;僅展開相關區段，減少捲動需求並改善載入時間。
- **觸控最佳化的標頭：**&#x200B;區段標頭易於點選和展開，支援行動裝置上的自然手勢。
- **平滑動畫：**&#x200B;展開和收合區段可提供使用者互動的視覺回饋。
- **空間效率：**&#x200B;摺疊的區段將垂直空間最小化，讓表單更容易在所有裝置上瀏覽。

**如何實作**

1. 將[Accordion元件](https://experienceleague.adobe.com/zh-hant/docs/experience-manager-core-components/using/adaptive-forms/adaptive-forms-components/accordion)新增至您的表單。
2. 在每個摺疊式功能表區段中，群組相關的選用或次要內容。
3. 針對每個區段使用清楚的描述性標頭，以協助使用者瞭解內含的資訊。
4. 根據重要性和使用者需求，為每個區段設定適當的預設開啟或關閉狀態。

**行動裝置優勢**

- 收合未使用的區段來減少捲動，讓使用者一次只專注於一個區段。
- 觸控互動功能支援自然的展開/收合手勢。
- 載入更快，因為只顯示作用中的內容。
- 改善焦點，因為使用者在任何指定時間都只會看到他們需要的資訊。

**最佳實務**

- 使用清除區段標題，讓使用者在展開區段之前先瞭解所需。
- 依邏輯將每個區段內的相關內容分組，以增進理解。
- 設定重要區段，以便在需要立即注意時開始展開。
- 提供簡短區段預覽或摘要，協助使用者決定要展開哪些區段。

**常見使用案例**

- **產品組態：**&#x200B;基本選項、進階設定、配件及支援的區段。
- **常見問題集Forms：**&#x200B;帳戶、帳單、技術和一般問題的群組。
- **設定Forms：**&#x200B;隱私權、通知、外觀和進階選項的區段。


## 第 3 部分：回應式設計最佳做法

### 各個裝置類型的最佳做法

+++行動裝置最佳化(320px-767px)

**佈局和互動：**

- 針對所有表單內容使用單欄版面配置，以最大化可讀性和易用性。
- 確保所有按鈕和互動元素的高度均至少為44畫素，以便進行可靠的觸控互動。
- 提供清晰、簡單的導覽，並包含可見的「上一步」和「下一步」按鈕。
- 將長表格細分，將每個區段中的捲動需求降到最低。
- 自動聚焦於第一個輸入欄位，以提示行動鍵盤。

**欄位指導方針：**

- 文字欄位應橫跨熒幕全寬，且具有足夠的觸控輸入邊框間距。
- 使用原生下拉式清單/選取元素，以最佳化行動裝置可用性。
- 實作原生日期選擇器以維持一致的行動體驗。
- 讓檔案上傳區域變大，並清楚標示以方便存取。

+++

+++平板電腦最佳化(768px-1199px)

**配置與可用性：**

- 利用相關欄位的雙欄版面配置，以充分利用增加的熒幕空間。
- 以直向和橫向測試表單外觀和可用性。
- 針對觸控和滑鼠輸入進行設計，確保所有控制項都能輕鬆存取。
- 增加內容區域大小，同時維持清晰的視覺階層和可讀性。

+++

+++桌上型電腦最佳化(1200px+)

**進階功能與配置：**

- 使用多欄版面配置以有效使用水準空間並減少垂直捲動。
- 提供鍵盤快速鍵以支援進階使用者，方便使用者經常執行動作。
- 實作互動式元素的暫留狀態和視覺化意見回饋。
- 提供進階驗證，包含複雜表單的清晰、詳細錯誤訊息。

## 疑難排解

### 版面配置問題

+++行動裝置上的表單版面分隔符號

**可能的原因：**

- 無法適應較小熒幕的固定寬度元素
- 覆寫行動樣式的案頭優先CSS
- 影像或內容溢位其容器

**如何修正：**

- 請確定所有影像和容器都使用相對或基於百分比的調整大小。
- 從行動優先的CSS方法和大型熒幕的圖層增強功能開始。
- 使用裝置模擬器和實際裝置來測試表單。
- 避免固定尺寸；使用彈性版面配置。

+++

+++觸控目標太小

**可能的原因：**

- 小於44px x 44px的按鈕或連結
- 互動式元素距離太靠近
- 自訂CSS減少預設觸控目標大小

**如何修正：**

- 確保每個互動式元素都至少為44px x 44px。
- 在按鈕、連結和其他控制項之間增加足夠的間距。
- 使用真正的觸控裝置進行測試，而不只是滑鼠。
- 視需要擴充觸控目標區域以提供協助工具。

+++

+++內容溢位問題

**可能的原因：**

- 不會換行長文字或標籤
- 寬度固定的容器
- 未回應式縮放的影像

**如何修正：**

- 啟用所有標籤和內容的文字繞排。
- 使用隨容器縮放的回應式影像。
- 設計可適應不同內容長度的彈性版面配置。
- 使用短內容和長內容進行測試，以確保適應性。

### 效能問題

+++在行動裝置上載入緩慢

**可能的原因：**

- 未最佳化的大型影像
- JavaScript沈重或過度
- 同時載入太多表單欄位

**如何修正：**

- 針對行動裝置最佳化影像並使用適當的檔案格式。
- 延遲或延遲載入非關鍵內容。
- 儘可能減少使用協力廠商指令碼和Widget。
- 簡化表單欄位，僅載入必要的專案。

+++

### 測試和驗證問題

+++模擬器與真實裝置的差異

**可能的原因：**

- 瀏覽器轉譯引擎的差異
- 滑鼠無法精確模擬觸控互動
- 網路速度差異

**如何修正：**

- 除了模擬器之外，請一律在真實裝置上進行測試。
- 使用多種瀏覽器和裝置進行全面測試。
- 模擬各種網路速度，找出效能瓶頸。
- 從目標對象中的實際使用者收集意見回饋。
