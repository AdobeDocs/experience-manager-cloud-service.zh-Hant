# 設定感謝訊息和重新導向 URL

提交後的體驗會顯著影響使用者滿意度和表單完成率。Adobe 的通用編輯器提供全方位的選項，方便設定使用者提交表單後所看到的內容，無論是透過個人化的感謝訊息，或是策略性地重新導向至特定頁面。

本文將提供實施感謝訊息及重新導向 URL 兩者的詳細指引，包括技術考量、最佳做法和使用者體驗準則，讓表單提交達到最大成效。

## 先決條件

在設定提交後體驗之前，請確保您具備以下條件：

**技術設定：**

- 擁有存取通用編輯器的適當權限
- 在通用編輯器中建立的現有自適應表單
- 了解組織的重新導向 URL 需求

**規劃的考量：**

- **訊息策略**：定義感謝訊息中要使用的語氣、內容長度以及要包含的具體資訊
- **重新導向策略**：確定目標頁面並確保針對表單填寫後的體驗進行最佳化
- **分析整合**：規劃如何追蹤使用者與感謝訊息或重新導向目標的互動

## 設定感謝訊息

感謝訊息可以立即確認表單提交成功，並可以包含個人化內容、後續步驟，或與使用者提交相關的重要資訊。

### 使用感謝訊息的時機

在以下情況下，使用感謝訊息的成效最好：

- **簡單確認**：使用者需要確認而且沒有其他導覽需求
- **指導性內容**：您需要提供具體的後續步驟或重要資訊
- **品牌一致性**：可以按照組織的溝通風格，精心設計相關訊息
- **單頁體驗**：使用者應停留在目前頁面，以保持工作流程的連續性

### 實施步驟

**1. 存取表單屬性**

在通用編輯器中開啟自適應表單，然後在工具列中按一下「**編輯表單屬性**」圖示。接著會開啟全方位表單屬性對話框。

![表單屬性圖示](/help/forms/assets/ue-form-properties-icon.png)

**2. 導覽至感謝設定**

在「表單屬性」對話框中，選取「**感謝**」標籤來存取提交後設定選項。

![通用編輯器表單屬性](/help/forms/assets/ue-form-properties.png)

**3. 設定訊息顯示**

從可用選項中選取&#x200B;**顯示訊息**。

![感謝您](/help/edge/docs/forms/universal-editor/assets/thankyou-ue.png)

**4. 建立您的訊息內容**

在「**訊息內容**」欄位中，使用 RTF 文字編輯器撰寫您的感謝訊息。編輯器支援：

- **文字格式**：粗體、斜體、底線和顏色選項
- **清單**：使用項目符號和編號清單來整理資訊
- **連結**：相關資源或後續步驟的直接連結
- **全螢幕編輯**：按一下「展開」圖示來獲得更大的編輯工作區

### 技術考量

**訊息顯示行為：**

- 表單提交後，立即用覆蓋式模態視窗顯示訊息
- 內容支援 HTML 格式並保持回應式設計
- 訊息可以由使用者關閉或是設定自動關閉計時器

**內容準則：**

- 在提供必要資訊的同時保持資訊簡潔明瞭
- 在適當的時候加入明確的後續步驟
- 考量加入參照編號或確認詳細資訊
- 確保格式適合行動裝置顯示

### 實施範例

    感謝您提交表單！
    
    已收到您的申請，並指派參照編號 #REF-2024-001234。
    
    **接下來會發生什麼事：**
    - 您將在 15 分鐘內收到一封確認電子郵件
    - 我們的團隊將在 2 個工作天內審閱您提交的內容
    - 如果我們需要更多資訊，將直接聯繫您
    
    **需要幫助？**&#x200B;請聯絡我們的支援團隊：support@example.com

## 設定重新導向 URL

重新導向 URL 會在表單提交後自動將使用者導覽至特定頁面，能夠與現有工作流程緊密整合，或將使用者導向至相關內容。

### 使用重新導向 URL 的時機

重新導向 URL 最適合：

- **工作流程整合**：將使用者導向至儀表板、帳戶頁面，或程序中的後續步驟
- **內容傳遞**：根據表單回覆顯示相關產品、服務或資訊
- **分析追蹤**：導向至實施特定追蹤設定的頁面
- **多步驟程序**：引導使用者進入複雜工作流程的下一階段

### 實施步驟

**1. 存取表單屬性**

在通用編輯器中開啟自適應表單，然後按一下「**編輯表單屬性**」圖示，開啟表單設定對話框。

![表單屬性圖示](/help/forms/assets/ue-form-properties-icon.png)

**2. 導覽至感謝設定**

選取「表單屬性」對話框中的「**感謝**」標籤，存取重新導向設定選項。

![通用編輯器表單屬性](/help/forms/assets/ue-form-properties.png)

**3. 啟用重新導向功能**

從可用的提交後選項中選擇「**重新導向至 URL**」。

![重新導向](/help/edge/docs/forms/universal-editor/assets/redirect-ue.png)

**4. 設定目標 URL**

在所提供的欄位中輸入目標 URL。系統支援多種 URL 格式，讓您享有實施的彈性。

### URL 設定選項

**絕對 URL**

完整的網址，包括通訊協定和網域：

    https://www.example.com/thank-you
    https://dashboard.example.com/user/profile

**相對路徑**

相對於目前網域的路徑：

    /thank-you
    /dashboard/user-profile
    ../confirmation-page.html

**AEM Sites 頁面參照**

參照至您的 AEM Sites 實施中其他頁面：

    /content/mysite/en/thank-you
    /content/mysite/en/next-steps

### 技術考量

**重新導向行為：**

- 表單提交成功後立即重新導向
- 瀏覽器歷史記錄包括重新導向的資訊，以便返回按鈕的功能能夠正常運作
- 重新導向的時間設定可以選擇搭配延遲

**URL 驗證：**

- 系統會在允許設定之前驗證 URL 格式
- 相對 URL 是根據目前網域進行解析
- 外部 URL 需要正確的 CORS 設定 (如需要)

## 最佳做法與建議

### 使用者體驗準則

**訊息最佳化：**

- **清楚明確最重要**：確保使用者能立即了解已經成功提交
- **增加價值**：提供相關資訊協助使用者進行後續步驟
- **維持一致的品牌風格**：維持組織的語氣和視覺風格
- **行動裝置考量**：在各種尺寸的螢幕上測試訊息

**重新導向最佳化：**

- **頁面最佳化**：確保重新導向目標針對提交表單後的訪客進行最佳化
- **載入效能**：驗證重新導向頁面能快速載入，以維持使用者體驗
- **內容關聯性**：確保重新導向內容與表單內容相關

### 安全性考量

**URL 驗證：**

- 對重新導向 URL 實施適當的驗證，防止惡意的重新導向
- 考量採用允許清單的方法指定允許重新導向的網域
- 監視重新導向模式是否存在異常活動

**內容安全性：**

- 清理感謝訊息內容，防止指令碼注入
- 針對 RTF 文字內容實施適當的內容安全性原則
- 定期對重新導向目標進行安全性審閱

### 分析與追蹤

**實施考量：**

- **目標追蹤**：設定感謝訊息瀏覽數和重新導向完成數的分析目標
- **使用者歷程對應**：追蹤使用者與表單提交後體驗的互動情況
- **轉換最佳化**：對不同的感謝訊息和重新導向目標進行 A/B 測試

**測量策略：**

- 監視頁面關閉前停留檢視感謝訊息的時間
- 追蹤感謝訊息中連結的點進率
- 分析重新導向目標頁面上的使用者行為

## 驗證查核點

設定好表單提交後體驗之後：

**設定驗證：**

- 表單屬性正確顯示所選取的感謝選項
- 預覽模式下訊息內容正常顯示
- 重新導向 URL 格式正確且可存取
- 訊息中的所有連結均正常運作

**使用者體驗測試：**

- 提交測試表單，驗證是否正確顯示感謝訊息
- 使用不同的瀏覽器測試重新導向功能
- 驗證感謝訊息的行動裝置回應能力
- 確認重新導向目標正確載入

**分析設定：**

- 感謝訊息的追蹤程式碼正確實施
- 已設定重新導向目標追蹤
- 目標完成事件正確觸發

## 後續步驟

成功設定好您的提交後體驗之後：

- **監視效能**：審閱分析資料，了解使用者對感謝訊息或重新導向頁面的參與度
- **反覆修正和改進**：利用使用者回饋和資料洞察改進提交後策略
- **規模實施**：將成功的模式套用至組織中其他表單
