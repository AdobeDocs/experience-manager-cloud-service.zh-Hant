# 設定感謝訊息和重新導向URL

提交後的體驗會大幅影響使用者滿意度和表單完成率。 Adobe的Universal Editor提供全方位的選項，可設定使用者在提交表單後看到的內容，不論是透過個人化的感謝訊息或策略性重新導向至特定頁面。

本文提供實作感謝訊息和重新導向URL的詳細指引，包括技術考量、最佳實務和使用者體驗指引，以最大化表單提交的成效。

## 先決條件

在設定提交後體驗之前，請確定您具備：

**技術設定：**

- 使用適當許可權存取通用編輯器
- 在通用編輯器中建立的現有最適化表單
- 瞭解貴組織的重新導向URL需求

**規劃考量事項：**

- **訊息策略**：定義要包含在感謝訊息中的語調、長度和特定資訊
- **重新導向策略**：識別目標頁面，並確定已針對表單後置完成體驗最佳化
- **Analytics整合**：規劃如何透過感謝訊息或重新導向目的地來追蹤使用者的互動

## 設定感謝訊息

感謝訊息會立即確認表單提交成功，並包含個人化內容、後續步驟或與使用者提交相關的重要資訊。

### 何時使用感謝訊息

在下列情況下，感謝訊息的運作方式最佳：

- **簡單確認**：使用者需要確認，不需要額外的導覽需求
- **教學內容**：您必須提供特定的後續步驟或重要資訊
- **品牌一致性**：訊息可以精心製作，以符合您組織的通訊風格
- **單頁體驗**：使用者應停留在目前頁面上，以保持工作流程連續性

### 實作步驟

**1. 存取表單屬性**

在通用編輯器中開啟最適化表單，然後按一下工具列中的&#x200B;**編輯表單屬性**&#x200B;圖示。 這會開啟完整的表單屬性對話方塊。

**2. 瀏覽至感謝您的設定**

在[表單內容]對話方塊中，選取&#x200B;**感謝您**&#x200B;索引標籤以存取提交後組態選項。

**3。 設定訊息顯示**

從可用選項中選取&#x200B;**顯示訊息**。 這會啟動具有RTF功能的訊息內容編輯器。

**4。 建立您的訊息內容**

在&#x200B;**訊息內容**&#x200B;欄位中，使用RTF編輯器製作您的感謝訊息。 編輯器支援：

- **文字格式**：粗體、斜體、底線和色彩選項
- **清單**：用於組織資訊的專案符號與編號清單
- **連結**：相關資源的直接連結或後續步驟
- **全熒幕編輯**：按一下展開圖示，以取得較大的編輯工作區

### 技術考量

**訊息顯示行為：**

- 訊息在提交表單後會立即顯示在強制回應覆蓋中
- 內容支援HTML格式並維持回應式設計
- 使用者可以解除訊息，或設定自動關閉計時器

**內容指南：**

- 提供必要資訊的同時保持簡潔
- 視需要包含明確的後續步驟
- 考慮包括參考編號或確認詳細資訊
- 確定適合行動裝置的格式

### 實作範例

    感謝您的提交！
    
    已收到您的應用程式，並已指派參考號碼#REF-2024-001234。
    
    **後續情形：**
     — 您會在15分鐘內收到確認電子郵件
     — 我們的團隊會在2個工作天內檢閱您的提交內容
     — 如需其他資訊，我們會直接聯絡您
    
    **需要協助?**請透過support@example.com聯絡我們的支援團隊

## 設定重新導向URL

在提交表單後，重新導向URL會自動將使用者導覽至特定頁面，實現與現有工作流程的無縫整合，或將使用者導向相關內容。

### 何時使用重新導向URL

重新導向URL最適合用於：

- **工作流程整合**：將使用者導向到儀表板、帳戶頁面或程式中的後續步驟
- **內容傳遞**：根據表單回應，展示相關產品、服務或資訊
- **Analytics追蹤**：導向具有特定追蹤實作的頁面
- **多步驟流程**：將使用者移至複雜工作流程的下一個階段

### 實作步驟

**1. 存取表單屬性**

在通用編輯器中開啟最適化表單，然後按一下&#x200B;**編輯表單屬性**&#x200B;圖示以開啟表單設定對話方塊。

**2. 瀏覽至感謝您的設定**

在[表單屬性]對話方塊中選取&#x200B;**感謝您**&#x200B;索引標籤，以存取重新導向組態選項。

**3。 啟用重新導向功能**

從可用的後送選項中選擇&#x200B;**重新導向至URL**。

**4。 設定目的地URL**

在提供的欄位中輸入您的目標URL。 此系統支援多種URL格式，以彈性實施。

### URL設定選項

**絕對URL**

完整的網址，包括通訊協定和網域：

    https://www.example.com/thank-you
    https://dashboard.example.com/user/profile

**相對路徑**

相對於您目前網域的路徑：

    /thank-you
    /dashboard/user-profile
    ../confirmation-page.html

**AEM Sites頁面參考**

對您AEM Sites實作中其他頁面的參照：

    /content/mysite/en/thank-you
    /content/mysite/en/next-steps

### 技術考量

**重新導向行為：**

- 重新導向會在成功提交表單後立即發生
- 瀏覽器記錄包含重新導向，以提供適當的返回按鈕功能
- 重新導向計時可設定為選擇性延遲

**URL驗證：**

- 在允許設定之前，系統會驗證URL格式
- 相對URL會針對目前的網域進行解析
- 如有需要，外部URL需要正確的CORS設定

## 最佳實務和建議

### 使用者體驗准則

**訊息最佳化：**

- **先釐清**：確保使用者立即瞭解他們的提交已成功完成
- **加值**：提供協助使用者後續步驟的資訊
- **一致的品牌**：維持您組織的聲音和視覺風格
- **行動考量**：測試各種熒幕大小的訊息

**重新導向最佳化：**

- **頁面最佳化**：確定已針對表單後的訪客最佳化重新導向目的地
- **載入效能**：確認重新導向頁面可快速載入，以維持使用者體驗
- **內容關聯性**：請確定重新導向內容與表單內容相關

### 安全性考量

**URL驗證：**

- 對重新導向URL實作適當的驗證，以防止惡意重新導向
- 考慮將允許的重新導向網域列入白名單方法
- 監控異常活動的重新導向模式

**內容安全性：**

- 處理感謝訊息內容，以防止指令碼插入
- 為RTF內容實作適當的內容安全性原則
- 定期安全性審查重新導向目的地

### 分析與追蹤

**實作考量事項：**

- **目標追蹤**：設定感謝訊息檢視和重新導向完成的分析目標
- **使用者歷程對應**：追蹤使用者與提交後體驗的互動情形
- **轉換最佳化**： A/B測試不同感謝訊息和重新導向目的地

**測量策略：**

- 監視刪除前感謝訊息所花費的時間
- 追蹤感謝訊息中連結的點進率
- 分析重新導向目的地頁面上的使用者行為

## 驗證查核點

設定您的提交後體驗後：

**組態驗證：**

- 表單屬性正確顯示選取的感謝選項
- 訊息內容在預覽模式中正確顯示
- 重新導向URL的格式正確且可存取
- 訊息內的所有連結皆可正常運作

**使用者體驗測試：**

- 提交測試表單，以驗證是否正確顯示感謝訊息
- 跨不同瀏覽器測試重新導向功能
- 驗證感謝訊息的行動回應速度
- 確認重新導向目的地已正確載入

**Analytics設定：**

- 已正確實作感謝訊息的追蹤程式碼
- 已設定重新導向目的地追蹤
- 目標完成事件已正確引發

## 後續步驟

成功設定您的後續提交體驗後：

- **監視效能**：檢閱分析以瞭解使用者參與感謝訊息或重新導向頁面
- **反複和改進**：使用使用者意見反應和資料深入分析來調整您的提交後策略
- **擴展實作**：將成功的模式套用至組織中的其他表單
