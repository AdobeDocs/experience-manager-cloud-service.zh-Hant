# 驗證的常用規則運算式

以下是一些可用來增強表單驗證的規則運算式，這些運算式超越了現代瀏覽器所提供的功能：

## 增強式密碼

```regex
^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)(?=.*[@$!%*?&])[A-Za-z\d@$!%*?&]{8,}$
```

確保使用至少8個字元：

* 小寫字母(a-z)
* 大寫字母(A-Z)
* 數字(0-9)
* 特殊字元(@$！%*?&amp;)


## 電子郵件地址


```regex
^[a-zA-Z0-9.!#$%&'*+/=?^_`{|}~-]+@[a-zA-Z0-9-]+(?:\.[a-zA-Z0-9-]+)*$
```

允許使用者名稱和網域名稱中包含字母、數字和特殊字元。


## 電話號碼（美國格式）

```regex
^\(?([0-9]{3})\)?[-. ]([0-9]{3})[-. ]([0-9]{4})$
```

以(XXX) XXX-XXXX格式驗證電話號碼。



## URL

```regex
^(http|https)://.*$
```

確保以http或https開頭的有效URL。



## 日期(YYYY-MM-DD)

```regex
^\d{4}-(0[1-9]|1[0-2])-(0[1-9]|[12][0-9]|3[01])$
```

驗證YYYY-MM-DD格式的日期。


## 時間(HH：MM)

```regex
^([01][0-9]|2[0-3]):[0-5][0-9]$
```

以HH：MM （24小時格式）驗證時間。


## 郵遞區號（美國格式）

```regex
^\d{5}(?:[-\ ]\d{4})?$
```

使用選用的連字型大小和4位數副檔名驗證5位數的美國郵遞區號。


## 使用者名稱（英數字元和底線）

```regex
^[a-zA-Z0-9_]+$
```

允許使用字母、數字和底線。


## 顏色十六進位程式碼

```regex
^#[0-9a-fA-F]{6}$
```

驗證6位數的十六進位色彩代碼。 例如， #FFFFFF。


## IP 位址

```regex
^(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})\.(25[0-5]|2[0-4][0-9]|1[0-9][0-9]|[0-9]{2,3})$
```

驗證IPv4位址。



## 社會安全號碼（美國格式）

```regex
^\d{3}-\d{2}-\d{4}$
```



## 信用卡號碼

```regex
^(?:4[0-9]{12}(?:[0-9]{3})?|[25][1-7][0-9]{14}$
```

以(XXX) XXX-XXXX格式驗證電話號碼。

