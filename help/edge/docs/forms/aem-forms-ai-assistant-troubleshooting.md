# Forms Experience Builder — 疑難排解指南

>[!NOTE]
>
> 率先採用者程式下提供Forms Experience Builder 。 從您的工作地址傳送電子郵件給`aem-forms-ea@adobe.com`以要求存取權。

>[!IMPORTANT]
>
> **檔案可能變更**：此疑難排解指南目前正在針對產品進行測試，可能會受到更新和修訂的影響。 隨著Forms Experience Builder在早期採用者計畫中持續演化，問題、解決方案和偵錯技術可能會發生變化。

此全方位的疑難排解指南可協助您識別、診斷及解決使用Forms Experience Builder時的常見問題。 本指南依問題類別組織，提供快速修正和詳細解決方案。

## 快速參考 — 常見問題

| 問題 | 快速修正 |
|-------|-----------|
| **介面未載入** | 重新整理瀏覽器、檢查網際網路連線、驗證早期存取許可權 |
| **命令無法運作** | 請嘗試`/help`或使用自然語言而非斜線命令 |
| **@fieldName無法辨識** | 檢查拼字，確定欄位存在最前，驗證欄位名稱語法 |
| **檔案上傳失敗** | 在10MB以下使用PDF/JPG/PNG，檢查檔案格式相容性 |
| **表單看起來錯誤** | 更明確些：「使其支援行動裝置」而非「修正版面」 |
| **整合失敗** | 驗證API認證和許可權，檢查端點可用性 |
| **表單未提交** | 檢查提交動作設定和驗證規則 |
| **驗證錯誤未顯示** | 驗證欄位驗證設定和錯誤訊息位置 |
| **行動版面配置問題** | 檢閱回應式設計設定和欄位大小 |
| **欄位未出現** | 檢查條件邏輯和可見性規則 |
| **匯入失敗** | 驗證檔案格式相容性和大小限制 |
| **效能問題** | 最佳化欄位計數並移除不必要的驗證 |
| **協助工具問題** | 檢閱欄位標籤、ARIA屬性和索引標籤順序 |

**仍需要協助嗎？**&#x200B;型別`/help`，後面接著您的特定問題，或連絡您的系統管理員以取得技術協助。

## 存取與驗證問題

### 無法存取Forms Experience Builder

**症狀：**

- Forms Experience Builder介面不可見
- 「拒絕存取」或類似的錯誤訊息
- 編輯器中缺少Forms Experience Builder圖示

**解決方案：**

1. **驗證早期存取方案註冊**
   - 確認您已獲得早期採用者計畫的核准
   - 檢查您的請求是否從您的正式工作電子郵件傳送
   - 若存取權仍擱置中，請連絡`aem-forms-ea@adobe.com`

2. **檢查環境設定**
   - 確認您的環境已啟用AEM Forms
   - 確定您使用支援的瀏覽器(Chrome、Firefox、Safari、Edge)
   - 清除瀏覽器快取和Cookie
   - 停用可能干擾此功能的瀏覽器延伸模組

3. **驗證使用者許可權**
   - 確認您擁有適當的使用者角色和許可權
   - 向您的系統管理員查詢存取許可權
   - 確認您使用正確的帳戶登入

### 介面載入問題

**症狀：**

- 空白或部分載入的介面
- 正在旋轉載入未完成的指示器
- 瀏覽器主控台中的JavaScript錯誤

**解決方案：**

1. **瀏覽器疑難排解**
   - 重新整理頁面（Ctrl+F5或Cmd+Shift+R）
   - 嘗試不同的瀏覽器或無痕模式/私用模式
   - 檢查瀏覽器更新，並安裝（如果可用）
   - 暫時停用廣告封鎖程式和隱私權擴充功能

2. **網路連線**
   - 驗證穩定的網際網路連線
   - 檢查公司防火牆是否封鎖必要的網域
   - 儘可能使用不同的網路連線進行測試
   - 連絡IT支援以解決網路組態問題

3. **快取和儲存問題**
   - 清除瀏覽器快取和本機儲存空間
   - 將瀏覽器設定重設為預設值
   - 檢查裝置上的可用磁碟空間
   - 嘗試從其他裝置存取

## 命令和互動問題

### 斜線命令無法運作

**症狀：**

- 無法辨識`/create-form`或其他斜線命令
- 未出現任何自動完成建議
- 命令會產生錯誤訊息

**解決方案：**

1. **命令語法驗證**
   - 確定正確的命令格式： `/command-name description`
   - 檢查命令名稱中的拼寫錯誤
   - 使用自然語言作為替代方法：「建立聯絡人表單」
   - 嘗試`/help`以驗證命令的可用性

2. **內容特定命令**
   - 確認您處於正確的編輯器內容(通用編輯器與最適化Forms編輯器)
   - 有些命令只適用於特定環境
   - 檢查命令參考以取得內容需求

3. **替代方法**
   - 使用自然語言而非斜線命令
   - 將複雜的命令分解成更小、更簡單的請求
   - 嘗試逐步建立表單，而非單一複雜命令

### 欄位參考無法運作

**症狀：**

- 無法辨識`@fieldName`個參考
- 有關未知欄位的錯誤訊息
- 欄位修改未正確套用

**解決方案：**

1. **欄位名稱驗證**
   - 檢查欄位名稱的確切拼字（區分大小寫）
   - 確認欄位存在再參考
   - 使用建立的確切欄位名稱，而不是顯示標籤
   - 驗證欄位命名慣例（駝峰式大小寫、蛇形大小寫等）

2. **欄位參考語法**
   - 請使用不含空格的正確`@fieldName`語法
   - 避免在欄位參考中使用特殊字元
   - 檢查隱藏的字元或格式問題
   - 嘗試手動重新建立欄位參考

3. **偵錯欄位參考**
   - 先列出所有現有欄位：「顯示所有目前表單欄位」
   - 在規則中參照欄位之前先建立欄位
   - 使用不含複雜字元的簡單欄位名稱
   - 測試欄位一次參照一個

## 表單建立與設計問題

### 表單未如預期建立

**症狀：**

- 產生的表單缺少請求的欄位
- 不正確的欄位型別或版面配置
- 表單結構不符合說明

**解決方案：**

1. **改善提示特殊性**
   - 在表單說明中更詳細
   - 指定確切的欄位型別和驗證需求
   - 包含版面偏好設定和使用者體驗需求
   - 將複雜的表單分解為較小的增量請求

2. **反複開發方法**
   - 從基本表單結構開始
   - 逐步新增欄位和功能
   - 在繼續之前測試每個新增
   - 透過對話而非單一複雜請求進行細化

3. **較佳提示範例**

   而非：

       建立客戶的表單
   
   使用：

       建立客戶連絡人表單，使用：
        — 全名（必要文字欄位）
        — 電子郵件地址（驗證時必填）
        — 電話號碼（選擇性，格式化）
        — 訊息（必要文字區域，最多500個字元）
        — 提交至電子郵件通知
   

### 版面配置和樣式問題

**症狀：**

- 行動裝置上的表單似乎損毀
- 間距或對齊方式不一致
- 欄位未正確顯示
- 不良視覺階層

**解決方案：**

1. **行動回應**
   - 請求行動裝置專屬最佳化：「讓此表單更適合行動裝置使用」
   - 指定回應式設計需求
   - 在實際行動裝置上測試
   - 對行動裝置使用單欄配置

2. **版面配置改善**
   - 詳細說明版面配置需求：「以兩欄排列位址列位」
   - 請求特定樣式：「使用專業色彩和簡潔的印刷樣式」
   - 指定間距和對齊需求
   - 要求協助工具合規性

3. **品牌一致性**
   - 在建立表單前準備品牌指南
   - 在要求中包含特定的色彩代碼和字型
   - 在所有表單中使用一致的樣式
   - 建立品牌範本以供重複使用

### 條件邏輯問題

**症狀：**

- 規則未如預期觸發
- 欄位顯示/隱藏不正確
- 驗證邏輯無法運作
- 複雜的商業規則失敗

**解決方案：**

1. **規則簡化**
   - 將複雜的規則分成更小、更簡單的條件
   - 組合前個別測試每個規則
   - 使用明確、特定的條件：「當@maritalStatus等於『已婚』時顯示@spouseInfo」
   - 一開始避免巢狀或過於複雜的邏輯

2. **規則測試和偵錯**
   - 測試所有可能的使用者路徑和情境
   - 驗證條件中的欄位名稱和值
   - 檢查規則條件是否區分大小寫
   - 使用偵錯模式來追蹤規則執行

3. **商務邏輯實作**
   - 實施前須清楚記錄業務需求
   - 逐步實作規則並測試每個步驟
   - 觸發規則時提供明確的使用者意見回饋
   - 處理邊緣案例和例外狀況

## 檔案匯入和轉換問題

### PDF匯入失敗

**症狀：**

- PDF檔案未上傳或處理
- 轉換後的表單缺少欄位或內容
- PDF轉換期間的錯誤訊息
- PDF的欄位認知度不佳

**解決方案：**

1. **檔案格式與大小**
   - 確認PDF檔案小於10MB
   - 使用高品質、文字型PDF （非掃描影像）
   - 確認PDF未受密碼保護或加密
   - 如果文字擷取失敗，請嘗試將PDF轉換為影像格式

2. **PDF品質最佳化**
   - 使用具有清晰、定義良好的表單欄位的PDF
   - 確保良好的對比和可讀文字
   - 避免複雜的版面或重疊的元素
   - 在轉換請求中提供其他內容

3. **轉換增強功能**
   - 詳細說明預期的表單結構
   - 指定欄位型別和驗證需求
   - 請求特定改善：「新增行動回應能力及驗證」
   - 手動檢閱並調整轉換後的表單

### 影像和熒幕擷圖轉換問題

**症狀：**

- 影像的欄位識別不佳
- 不正確的欄位型別或版面配置
- 遺失表單元素
- 轉換錯誤或逾時

**解決方案：**

1. **影像品質需求**
   - 使用高解析度影像（至少300 DPI）
   - 確保良好的光線和對比
   - 避免陰影、炫光或扭曲
   - 裁切影像以僅聚焦於表單內容

2. **最佳影像格式**
   - 使用PNG或JPG格式以獲得最佳結果
   - 避免GIF或低品質的壓縮影像
   - 確保影像中的文字可清楚讀取
   - 視需要嘗試不同的影像方向

3. **轉換指南**
   - 提供表單結構的詳細說明
   - 明確指定欄位型別和需求
   - 在轉換期間要求特定的增強功能
   - 準備在轉換後進行手動調整

## 整合與提交問題

### 表單提交失敗

**症狀：**

- Forms未成功提交
- 提交期間的錯誤訊息
- 資料未送達預期目的地
- 提交期間發生逾時錯誤

**解決方案：**

1. **提交動作組態**
   - 確認已正確設定提交動作
   - 檢查API端點和驗證認證
   - 先使用簡單的電子郵件提交進行測試
   - 驗證資料格式需求

2. **網路和連線**
   - 檢查網際網路連線和網路穩定性
   - 驗證防火牆設定允許表單提交
   - 從不同的網路連線測試
   - 檢查是否有公司Proxy或安全性限制

3. **資料驗證問題**
   - 確認所有必填欄位均已完成
   - 驗證資料格式符合API要求
   - 檢查特殊字元或編碼問題
   - 先使用最小資料集進行測試

### API整合問題

**症狀：**

- REST API端點沒有回應
- 驗證失敗
- 資料格式不相符
- 整合逾時或錯誤

**解決方案：**

1. **API組態驗證**
   - 驗證API端點URL是否正確且可存取
   - 檢查驗證認證和許可權
   - 使用Postman等工具獨立測試API端點
   - 驗證API是否接受正確的資料格式（JSON、XML等）

2. **資料對應問題**
   - 確保表單欄位名稱符合API引數要求
   - 檢查可能遺漏的必填欄位
   - 驗證資料型別相容性（字串、數字、日期）
   - 使用範例資料進行測試以識別對應問題

3. **處理及偵錯錯誤**
   - 啟用API呼叫的詳細錯誤記錄
   - 檢查API回應代碼和錯誤訊息
   - 針對暫時失敗實作重試邏輯
   - 在API失敗時為使用者提供後援選項

### 電子郵件整合問題

**症狀：**

- 確認電子郵件未傳送
- 電子郵件傳送到垃圾郵件資料夾
- 不正確的電子郵件格式
- 電子郵件中缺少表單資料

**解決方案：**

1. **電子郵件設定**
   - 驗證電子郵件地址的格式是否正確
   - 檢查SMTP設定和驗證
   - 先使用簡單的電子郵件地址進行測試
   - 驗證電子郵件伺服器許可權和配額

2. **電子郵件傳遞最佳化**
   - 使用適當的電子郵件標題和寄件者資訊
   - 避免在主題行中觸發垃圾訊息字詞
   - 包含適當的取消訂閱機制
   - 測試傳送給不同提供者的電子郵件

3. **內容和格式**
   - 驗證電子郵件中的表單資料格式是否正確
   - 檢查特殊字元或編碼問題
   - 測試包含各種資料組合的電子郵件範本
   - 確保電子郵件內容可供存取及讀取

## 效能和載入問題

### 緩慢表單載入

**症狀：**

- Forms最初需要很長時間才能載入
- 使用者互動緩慢
- 表單作業期間逾時
- 行動裝置上的效能不佳

**解決方案：**

1. **表單最佳化**
   - 減少欄位數量和複雜性
   - 對非關鍵區段實作延遲載入
   - 最佳化影像和資產，以利網頁傳遞
   - 移除不必要的驗證規則或邏輯

2. **瀏覽器和裝置最佳化**
   - 清除瀏覽器快取和暫存檔案
   - 關閉不必要的瀏覽器分頁和應用程式
   - 檢查可用的裝置記憶體與儲存空間
   - 嘗試不同的瀏覽器來比較效能

3. **網路最佳化**
   - 測試不同的網路連線
   - 檢查網路阻塞或頻寬限制
   - 儘可能使用有線連線而非WiFi
   - 如需網路效能問題，請聯絡IT支援

### 驗證效能問題

**症狀：**

- 驗證回應緩慢
- 延遲的錯誤訊息顯示
- 表單在驗證期間凍結
- 欄位驗證期間發生逾時錯誤

**解決方案：**

1. **驗證最佳化**
   - 減少即時驗證的頻率
   - 實作驗證呼叫的取消跳出
   - 簡化複雜的驗證規則
   - 儘可能使用使用者端驗證

2. **規則簡化**
   - 將複雜的驗證分成較小的規則
   - 移除不必要的跨欄位驗證
   - 針對效能最佳化條件式邏輯
   - 適當時快取驗證結果

3. **改善使用者體驗**
   - 提供簡易驗證的立即回饋
   - 對於複雜規則，請使用漸進式驗證，而非即時驗證
   - 在驗證程式期間顯示載入指標
   - 允許使用者在背景進行驗證時繼續

## 進階疑難排解

### 偵錯模式和診斷

**啟用偵錯資訊**

使用這些提示來取得有關表單問題的詳細資訊：

    啟用偵錯模式以識別表單提交和欄位驗證的問題
    
    分析表單錯誤：檢查驗證規則、API回應和使用者輸入模式
    
    顯示有關表單結構和欄位設定的詳細資訊

### 錯誤分析技術

**系統偵錯方法**

1. **隔離問題**
   - 以最少的表單設定進行測試
   - 暫時移除複雜功能
   - 個別測試個別元件
   - 使用排除程式來識別根本原因

2. **收集診斷資訊**
   - 檢查JavaScript錯誤的瀏覽器主控台
   - 檢閱網路要求與回應
   - 記錄重現問題的確切步驟
   - 收集熒幕擷取畫面和錯誤訊息

3. **測試環境變數**
   - 嘗試不同的瀏覽器和裝置
   - 使用不同的使用者帳戶和許可權進行測試
   - 在不同的網路環境中驗證
   - 與工作表單或設定比較

### 記錄分析與監視

**瀏覽器主控台偵錯**

1. 開啟瀏覽器開發人員工具(F12)
2. 檢查Console索引標籤以瞭解JavaScript錯誤
3. 檢閱網路索引標籤以找出失敗的請求
4. 針對緩慢作業監視「效能」標籤

**常見錯誤模式**

- **CORS錯誤**： API整合的跨原始要求問題
- **驗證失敗**：認證無效或權杖過期
- **驗證錯誤**：欄位驗證規則衝突或語法錯誤
- **網路逾時**：網路連線緩慢或不可靠
