# Moving to AEM as a Cloud Service {#aem-moving-to-aem-guide}

## 以雲端服務的形式改用AEM的優點 {#benefits-aem-moving-to-aem-guide}

AEM作為雲端服務，為Experience Manager Sites和Assets提供可擴充、安全且靈活的技術基礎，讓行銷人員和IT專注於大規模提供具影響力的體驗。

有了AEM作為雲端服務，您的團隊可以專注於創新，而非規劃產品升級。 新產品功能會經過徹底測試並傳遞給您的團隊，不會中斷，因此他們隨時都能存取最新的應用程式。

## 瞭解向雲服務過程的過渡 {#understanding-move}

向雲服務的過渡過程包括三個階段：規劃、執行和上線後。

為了順利順利順利地完成過渡，您應確保正確規劃並遵守本指南中概述的最佳實踐。

下圖顯示建議的雲端服務轉場歷程的圖片表示。

![影像](/help/move-to-cloud-service/assets/home-img1.png)

進一步瞭解雲服務過渡過程中涉及的不同階段：

* **[執行](/help/move-to-cloud-service/execution.md)**
* **[貼文上線](/help/move-to-cloud-service/post-go-live.md)**
