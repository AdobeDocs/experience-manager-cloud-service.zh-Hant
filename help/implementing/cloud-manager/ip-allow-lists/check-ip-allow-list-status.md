# 正在檢查IP允許清單狀態{#check-allow-list-status}

請依照下列步驟，判斷IP允許清單的更新狀態：

1. 從&#x200B;**Environments**&#x200B;螢幕按一下表格中「IP允許清單」的「狀態」表徵圖，然後選擇&#x200B;**IP允許清單**&#x200B;頁。

1. Cloud Manager會顯示下列其中一種狀態，如下節所述。

## IP允許清單的狀態 {#status}

以下是將顯示在IP允許清單中的狀態定義：

* **已套用**:環境上已成功套用IP允許清單。環境和服務可如下圖所示。

* **更新**:正在更新IP允許清單，該清單可能包括一個或多個應用或取消應用。每個「應用」和「取消應用」都將與「未啟動/正在進行/完成」或「失敗」一起列出。

* **失敗**:更新中的一個或多個應用或取消應用進程失敗。每個「應用」和「取消應用」都將與「完成」或「失敗」一起列出。
   * 即使更新中的一個應用/取消應用失敗，狀態仍為「失敗」。
   * 在清除所有故障之前，狀態將保持「失敗」狀態。 使用者必須選取狀態旁的重試圖示，才能清除失敗。
   * 狀態為「失敗」時，用戶將無法更新或刪除IP允許清單。

* **刪除**:正在刪除請求。這涉及取消應用所有服務。 每個「取消應用」都將與「未啟動/正在進行/完成」或「失敗」一起列出。
完成刪除操作後，IP允許清單將：
   * 「IP允許清單」表格中不再顯示。
   * 不再套用至Cloud Manager中程式中的任何服務。

* **刪除失敗**:刪除操作中的一個或多個取消應用進程失敗。每個「取消應用」都將與「完成」或「失敗」一起列出。

   * 即使取消應用失敗，狀態仍為「刪除失敗」。
   * 在清除所有故障之前，狀態將保持「Delete Failed（刪除失敗）」。 用戶必須從&#x200B;**中選擇刪除……**&#x200B;功能表，以清除任何失敗。
   * 當狀態為「失敗」時，用戶將不允許更新IP允許清單。

## IP允許清單的預先存在CDN設定{#pre-existing-cdn}

若環境中包含IP允許清單、SSL憑證或自訂網域名稱的現有CDN設定，則客戶會在&#x200B;**IP允許清單**&#x200B;和&#x200B;**環境**&#x200B;詳細資訊頁面中看到下列訊息。 當客戶透過UI完全移轉所有預先存在的環境設定後，UI上顯示的訊息就會消失，而訊息可能需要1到2個工作天才會消失。

>[!NOTE]
>若要查看及管理預先存在的設定，必須透過UI新增這些設定。 如需詳細資訊，請參閱[新增IP允許清單](/help/implementing/cloud-manager/ip-allow-lists/add-ip-allow-lists.md) 。
