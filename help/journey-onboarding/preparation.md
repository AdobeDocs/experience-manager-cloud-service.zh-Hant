# 登機準備 {#onboarding-preparation}

現在，您已通過閱讀文章瞭解了登機過程中準備的內容 [AEMas a Cloud Service登機旅程簡介，](overview.md) 你可能已經準備好了。 但是，在登錄過程開始之前，系統管理員在登錄系統之前必須瞭解一些數字或準備步驟。

## 目標 {#objective}

AEMas a Cloud Service是一種功能強大、靈活的工具，提供多種解決方案、附加模組和部署模型。 您的組織有權使用的解決方案在您與Adobe的合同中定義，該合同將指導入門流程的開始。

閱讀完此文檔後，您將瞭解

* 您與Adobe的合同定義什麼。
* 什麼Adobe規定。
* 該合同對登機過程的影響。
* 如何準備登機通知。

如果您不熟悉此頁上使用的所有術語，請不要擔心。 在下一段旅程中，他們會被解釋。

## 您的合同 {#contract}

開始使AEM用as a Cloud Service時，您的組織和Adobe之間簽訂了合同。 這是因為合同指定了Adobe與您的組織之間的若干重要協定，例如：

* 您的組織可以使用哪些解決方案和附加模組。
* 您可以訪問的環境的數量和類型
* 服務的開始日期。
* 您的卓越支援級別（如果適用）。
* 你的聯繫人，請登機。

如果您不瞭解這些詳細資訊，請與您組織內的承包小組聯繫。

簽訂合同後，Adobe將為您的環境提供服務，歡迎電子郵件將發送到合同中定義的登機聯繫人，並按Adobe組織登機呼叫。 除非安排未來服務開始日期，否則通常在合同結束後幾天內發生。

## 系統設定 {#provisioning}

一旦簽署合同，合同AEM所涵蓋的所有雲環境將由Adobe自動配置。 這些環境同樣會連結到Cloud Manager中的程式（在旅程的稍後部分有關此主題的詳細資訊）。 這AEM些雲環境包括在每AEM個as a Cloud Service訂閱中，通常由至少一個生產環境、一個分段環境以及可選的一個或多個開發或test環境組成。

## 登機呼叫 {#onboarding-call}

在合同完成時，Adobe登記小組安排與您組織的代表進行電話聯繫，討論如何開始。

您組織的與會者應包括：

* 業務線索
* 技術從業人員
* 您的實施合作夥伴

Adobe將由：

* 登機專家
* 您的技術客戶經理（如適用）
* 您指定的客戶成功工程師（如適用）
* 根據需要增加主題專家

此次電話會議的目標是讓您有機會提問、瞭解支援流程，並可能首次查看產品。

## 歡迎電子郵件 {#welcome-email}

在完成環境設定過程後，合同中定義的系統管理員將收到一封歡迎電子郵件，確認他們已被授予訪問Adobe Experience Cloud的權限。 如果你在閱讀此文檔旅程，那人很可能是你！

該電子郵件包含有關如何開始使用Experience Cloud服務和Cloud Manager自助服務門戶的詳細資訊。 此外，電子郵件還包含重要資訊，如支援資源、論壇、常見問題解答等。 在電子郵件中提供的資源清單中，您還將獲得有關如何訪問雲環境AEM的詳細資訊。

>[!TIP]
>
>如果您收到歡迎電子郵件且不是系統管理員，請在歡迎呼叫期間通知Adobe。 或者，您可以繼續此過程，並在後續步驟中學習如何親自將另一個同事指定為系統管理員。
